1. TESTING SETUP & PREREQUISITES
================================================================================

BACKEND SETUP:
--------------
1. Ensure MongoDB is running (local or cloud)
2. Set up .env file with:
   - MONGODB_URI
   - JWT_SECRET
   - EMAIL_USER & EMAIL_PASS (for Nodemailer)
   - PORT (default: 5000)
   - NODE_ENV=development

3. Install dependencies:
   cd backend
   npm install

4. Start backend server:
   npm run dev

FRONTEND SETUP:
---------------
1. Install dependencies:
   cd frontend
   npm install

2. Configure axios baseURL in frontend/src/services/axios.js

3. Start frontend server:
   npm run dev

TESTING TOOLS RECOMMENDED:
--------------------------
- Postman or Insomnia (API testing)
- Browser DevTools (Network tab, Console)
- MongoDB Compass (Database inspection)
- Chrome/Firefox (Cross-browser testing)

2. BACKEND API TESTING
================================================================================

2.1 AUTHENTICATION ROUTES (/auth)
----------------------------------

TEST CASE 1: User Login
- Endpoint: POST /auth/login
- Body: { "username": "admin", "password": "admin123", "rememberMe": true }
- Expected: 200 OK, JWT token in cookie, user object in response
- Check: firstLogin flag, role, user details

TEST CASE 2: Get Current User
- Endpoint: GET /auth/me
- Headers: Cookie with JWT token
- Expected: 200 OK, user profile with role-specific data
- For students: currentTeam should be populated
- For mentors: assignedTeams should be populated

TEST CASE 3: Logout
- Endpoint: POST /auth/logout
- Expected: 200 OK, token cookie cleared

TEST CASE 4: Change Password (First Login)
- Endpoint: PUT /auth/me/password
- Body: { "newPassword": "newPass123" }
- Expected: 200 OK, firstLogin set to false

TEST CASE 5: Forgot Password
- Endpoint: POST /auth/forgot-password
- Body: { "email": "user@example.com" }
- Expected: 200 OK, email sent with reset code

TEST CASE 6: Reset Password
- Endpoint: POST /auth/reset-password
- Body: { "email": "user@example.com", "code": "123456", "newPassword": "new123" }
- Expected: 200 OK, password updated

NEGATIVE TESTS:
- Login with wrong password → 401
- Login without username/password → 400
- Access /auth/me without token → 401
- Invalid reset code → 400
